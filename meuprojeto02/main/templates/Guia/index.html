Instalando o Django

Etapas

1. Instalar o venv

Primeiramente, você precisa instalar o virtualenv, uma ferramenta para criar ambientes virtuais Python isolados. Isso ajuda a manter as dependências do projeto separadas de outras instalações no seu sistema.

pip install virtualenv

2. Criar um Ambiente Virtual

Depois de instalar o virtualenv, você deve criar um novo ambiente virtual no diretório do seu projeto. Isso vai garantir que o Django e outras bibliotecas sejam instalados apenas nesse ambiente, sem interferir no seu sistema global.

python -m venv .venv

3. Ativar o Ambiente Virtual

Com o ambiente virtual criado, você precisa ativá-lo. Isso irá garantir que o Python e os pacotes instalados sejam isolados e específicos para esse projeto.

.\\.venv\\Scripts\\activate

4. Atualizar o pip

Agora, atualize o pip para garantir que você tenha a versão mais recente do instalador de pacotes Python.

python.exe -m pip install --upgrade pip

5. Instalar o Django

Com o ambiente virtual ativado e o pip atualizado, agora você pode instalar o Django. Use a versão que você deseja (exemplo: 5.0.6) ou a mais recente.

pip install Django==5.0.6

6. Verificar a instalação do Django

Após a instalação, verifique se o Django foi instalado corretamente executando o comando abaixo. Isso exibirá a versão do Django instalada.

django-admin --version

7. Instalar o Conector MySQL

Se você pretende usar o MySQL como banco de dados no Django, instale o conector necessário para conectar o Django ao MySQL.

pip install mysql-connector

8. Criar um novo Projeto Django

Agora, crie um novo projeto Django. O comando abaixo cria a estrutura básica de um projeto Django no diretório "meuprojeto".

django-admin startproject meuprojeto

9. Acessar o Diretório do Projeto

Depois de criar o projeto, entre no diretório do projeto que foi criado, usando o comando abaixo.

cd meuprojeto

10. Explorar a Estrutura do Projeto

Agora que o projeto foi criado, explore a estrutura de diretórios e arquivos gerados. Esses arquivos são fundamentais para o gerenciamento e funcionamento do seu projeto Django.

  • manage.py: Gerencia tarefas do Django, como migrações e testes.
  • meuprojeto/settings.py: Configurações globais do seu projeto.
  • meuprojeto/urls.py: Mapeamento das URLs para as views da aplicação.
  • meuprojeto/wsgi.py: Ponto de entrada para o servidor web do Django.

11. Criar um novo Aplicativo

O Django é baseado no conceito de aplicativos. Cada funcionalidade ou parte do seu site é dividida em aplicativos. O comando abaixo cria um novo aplicativo chamado "meu_app".

python manage.py startapp meu_app

12. Adicionar o Aplicativo ao `settings.py`

Agora, você precisa adicionar seu aplicativo ao arquivo de configurações do projeto (settings.py) para que ele seja reconhecido pelo Django.

            INSTALLED_APPS = [
                'django.contrib.admin',
                'django.contrib.auth',
                'django.contrib.contenttypes',
                'django.contrib.sessions',
                'django.contrib.messages',
                'django.contrib.staticfiles',
                'meu_app',
            ]

13. Configurar URLs do Aplicativo

Você precisa mapear as URLs do seu aplicativo para as views (funções que retornam respostas para as requisições HTTP). Isso é feito no arquivo urls.py do projeto.

            from meu_app import views
            urlpatterns = [
                path('', views.index, name='index'),
            ]

14. Criar Função de Visualização

Em seguida, crie uma função em views.py que será responsável por retornar uma resposta para o navegador (ex: renderizar um template HTML).

            from django.shortcuts import render
            def index(request):
                return render(request, 'index.html')

15. Instalar o `mysqlclient`

Se você optou por usar o MySQL como banco de dados, você precisará instalar o mysqlclient, um conector para o MySQL que permite a integração do Django com esse banco.

pip install mysqlclient

16. Realizar Migração para MySQL

Após configurar o banco de dados, execute a migração para aplicar as configurações e criar as tabelas necessárias no banco de dados.

python manage.py migrate

17. Criar Superusuário

Crie um superusuário para poder acessar o painel de administração do Django e gerenciar o conteúdo do seu site.

python manage.py createsuperuser

18. Configurar Banco de Dados no `settings.py`

No arquivo settings.py, configure as informações do seu banco de dados, como nome, usuário, senha, e o tipo de banco (MySQL, no caso).

            DATABASES = {
                'default': {
                    'ENGINE': 'django.db.backends.mysql',
                    'NAME': 'aula06', 
                    'USER': 'root',  
                    'PASSWORD': '',  
                    'HOST': '127.0.0.1', 
                    'PORT': '3306', 
                    'OPTIONS': {
                        'autocommit': True,
                    }
                }
            }

19. Iniciar o Servidor de Desenvolvimento

Finalmente, inicie o servidor de desenvolvimento para ver sua aplicação funcionando localmente. Acesse http://127.0.0.1:8000/ no navegador.

python manage.py runserver
